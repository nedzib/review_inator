# AGENTS.md

## Repo Shape
- This repo is just two Bash entrypoints: `pr_watcher.sh` (daemon logic) and `install.sh` (install/restart/uninstall for launchd). There is no package manager, test runner, linter, or CI config in the repo.

## Source Of Truth
- Trust the scripts over the README when they differ.
- `install.sh` installs `pr_watcher.sh` to `~/.local/bin/pr_watcher.sh`, not in-place from the repo. After changing `pr_watcher.sh`, run `./install.sh restart` or your edits will not affect the daemon.

## Runtime Files
- Repo files are code only. User state lives outside the repo:
- `~/.review_inator.config`: JSON config edited by the user.
- `~/.review_inator.log`: NDJSON history of processed PRs.
- `~/.review_inator.out.log` and `~/.review_inator.err.log`: launchd logs.
- `~/Library/LaunchAgents/com.nedzib.pr_watcher.plist`: generated by `install.sh`.

## Verified Commands
- Install and seed existing PRs: `./install.sh`
- Restart after code or config changes: `./install.sh restart`
- Uninstall launchd job and installed script: `./install.sh uninstall`
- Manual one-off seed without launching review sessions: `./pr_watcher.sh --seed`
- Check daemon status: `launchctl list | grep pr_watcher`
- Follow logs: `tail -f ~/.review_inator.out.log` and `tail -f ~/.review_inator.err.log`

## Behavior That Is Easy To Miss
- launchd runs the installed script through `/bin/zsh -l -c ...`; it depends on login-shell `PATH`.
- `pr_watcher.sh` hard-fails if `gh`, `git`, `jq`, `tmux`, or `osascript` are missing. `claude` is used later via `tmux send-keys` but is not prevalidated.
- Only PRs with a direct `requested_reviewer` matching the authenticated `gh` user are processed. Team review requests are explicitly skipped.
- New PR handling does all of this:
- `git fetch origin <branch>`
- `git worktree add --detach <parent>/<pr_number>_<github_user> origin/<branch>`, then `git switch` to the PR branch inside that worktree
- creates `.claude/settings.local.json` inside the worktree with Claude permissions set to `auto`
- writes `.pr_review_prompt` inside the worktree
- creates tmux session `<pr_number>_<github_user>`
- sends `claude "$(cat .pr_review_prompt)"` into that tmux session
- appends an NDJSON record to `~/.review_inator.log`
- `./install.sh` seeds currently open PRs before loading launchd, so only later PRs trigger worktrees/sessions on first install.
- Seeded entries are logged with `worktree` and `session` set to `seeded`.

## Editing Guidance
- Keep changes Bash-only and minimal; there is no surrounding framework.
- Be careful when changing path conventions or log schema: README examples and existing user data assume `~/.review_inator.*` files and NDJSON log lines.
